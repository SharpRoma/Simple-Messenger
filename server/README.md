# Simple Messenger — Серверная часть

Это серверная часть кроссплатформенного мессенджера. Сервер представляет собой асинхронное приложение на Python, использующее чистые TCP-сокеты с передачей данных в формате JSON. Сервер полностью автономен, работает в Docker-контейнере и автоматически управляет своей базой данных, медиафайлами и SSL-сертификатами.

## 🛠 Технологический стек
* **Язык:** Python 3.12+
* **Асинхронность:** `asyncio`
* **База данных:** SQLite3
* **Безопасность:** `ssl` (TLS-шифрование трафика), `bcrypt` (хеширование паролей)
* **Инфраструктура:** Docker, Docker Compose

## Установка и развертывание (VPS / Ubuntu)

### 1. Подготовка сервера
Убедитесь, что на вашем сервере установлены Docker и плагин Docker Compose.
```bash
sudo apt update
sudo apt install docker.io docker-compose-v2 -y
sudo systemctl enable --now docker
```

### 2. Перенос файлов на сервер
Скопируйте папку `server` с локального компьютера на VPS с помощью `scp`:
```bash
# Выполнять на локальном компьютере:
scp -r server/ root@IP_ВАШЕГО_СЕРВЕРА:~/messenger_server
```

### 3. Настройка переменных окружения
Подключитесь к VPS по SSH, перейдите в папку проекта и создайте файл `.env`. 
**Внимание:** Сервер не запустится без переменной `SERVER_SECRET`.
```bash
cd ~/messenger_server
echo "SERVER_SECRET=СюдаВашСекретныйКодДляРегистрации" > .env
```

### 4. Запуск сервера
Поднимите контейнер в фоновом режиме:
```bash
docker compose up -d --build
```

При первом запуске скрипт `entrypoint.sh` **автоматически сгенерирует самоподписанные SSL-сертификаты** сроком на 10 лет и создаст базу данных.

## Управление данными

Все критически важные данные хранятся в защищенном Docker-томе (`messenger_data`) и пробрасываются в контейнер по пути `/app/data`.
Даже если вы удалите контейнер командой `docker compose down`, данные **не удалятся**.

Внутри тома хранятся:
1. `messenger.db` — База данных пользователей, чатов и истории сообщений.
2. `cert.pem` и `key.pem` — Ключи шифрования SSL/TLS.
3. `uploads/` — Папка со всеми пересылаемыми медиафайлами (картинки, документы). *При удалении сообщения пользователем в клиенте, физический файл автоматически удаляется отсюда.*

## Полезные команды Docker

**Посмотреть логи сервера в реальном времени:**
```bash
docker logs -f simple_messenger_server
```

**Перезапустить сервер (например, после изменения `.env`):**
```bash
docker compose restart
```

**Остановить сервер:**
```bash
docker compose stop
```

**Остановить сервер и удалить контейнер (данные в томе сохранятся):**
```bash
docker compose down
```

**Проверить срок действия SSL-сертификата:**
```bash
docker exec simple_messenger_server openssl x509 -in /app/data/cert.pem -noout -enddate
```

## Протокол связи

Сервер слушает TCP-порт `8888`.
Ограничение на размер входящего пакета (буфер) установлено на **50 МБ** для поддержки передачи файлов.

Общение происходит в формате `JSON \n`. Каждое сообщение должно заканчиваться символом переноса строки.

**Базовый пакет отправки сообщения:**
```json
{
  "action": "send_msg",
  "chat_id": 1,
  "text": "Привет, мир!"
}
```

**Базовый пакет передачи файла:**
```json
{
  "action": "send_file",
  "chat_id": 2,
  "filename": "image.png",
  "data": "<Base64 Строка>"
}
```

## Безопасность

* **Шифрование:** Весь трафик (включая авторизацию и передачу файлов) обернут в контекст `ssl.create_default_context(ssl.Purpose.CLIENT_AUTH)`.
* **Регистрация:** Закрыта от случайных пользователей в интернете. Создать аккаунт можно только передав правильный параметр `secret` в пакете авторизации, который сверяется с переменной `.env` на сервере.
* **Пароли:** Хранятся в SQLite исключительно в виде солированных хешей `bcrypt`. Исходный пароль восстановить невозможно даже при утечке дампа БД.
