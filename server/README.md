Современный, асинхронный и высоконагруженный бэкенд для **Simple Messenger**. 
Написан на базе фреймворка **FastAPI** с использованием **PostgreSQL** в качестве основной базы данных. Архитектура сервера строго разделена на слои, что обеспечивает легкость поддержки, масштабируемость и безопасность.

## Ключевые возможности

* **Полная асинхронность (asyncio):** Сервер не блокируется при чтении из базы данных или сохранении файлов.
* **REST API + WebSockets:** Гибридная архитектура. Тяжелые операции (авторизация, история, файлы) обрабатываются по протоколу HTTP (REST), а мгновенная доставка сообщений работает через легковесные WebSockets.
* **Стриминг файлов:** Передача медиафайлов происходит потоково. Загрузка файла размером 10 ГБ потребует не более 1 МБ оперативной памяти (ОЗУ) сервера.
* **Swagger UI:** Автоматическая генерация OpenAPI-спецификации. Доступна прямо в браузере по адресу `/docs`.
* **Продвинутая безопасность:** 
  * Аутентификация через JWT-токены (`HS256`).
  * Пароли хешируются с использованием `bcrypt` и соли.
  * Регистрация защищена общим `SERVER_SECRET` кодом.
* **Ролевая система:** Автоматическое создание суперпользователя `admin` с правами на глобальное удаление любых сообщений.

## Структура проекта

Проект построен на FastAPI:

```text
server/
├── api/                   # Слой контроллеров (Endpoints)
│   ├── dependencies.py    # Зависимости (проверка JWT-токенов, прав)
│   ├── websockets.py      # ConnectionManager для реалтайм-сессий
│   └── routes/            # Маршруты (auth, chats, messages, chat_ws)
├── core/                  # Слой ядра
│   ├── config.py          # Валидация переменных окружения (.env) через Pydantic
│   ├── database.py        # Инициализация SQLAlchemy и asyncpg
│   └── security.py        # Логика хеширования паролей и генерации токенов
├── models/                # Слой данных (SQLAlchemy ORM модели)
│   ├── base.py            # Базовый декларативный класс
│   └── user, chat, msg... # Таблицы БД
├── schemas/               # Слой схем (Pydantic)
│   └── ...                # Валидация входящих/исходящих данных и генерация Swagger
├── docker-compose.yml     # Файл для быстрого деплоя (Production)
├── Dockerfile             # Сборка образа сервера
├── run.py                 # Скрипт для локального запуска (Development)
└── .env.example           # Шаблон конфигурационного файла
```

## Быстрый старт

Для развертывания сервера на вашем VDS используется готовый Docker-образ из GitHub Container Registry (GHCR).

### 1. Подготовка сервера
Убедитесь, что на сервере установлены `docker` и `docker-compose`.
Создайте директорию для мессенджера и перейдите в нее:
```bash
mkdir simple-messenger && cd simple-messenger
```

### 2. Настройка окружения
Скачайте файлы `docker-compose.yml` и `.env.example` из этого репозитория:
```bash
wget https://raw.githubusercontent.com/SharpRoma/Simple-Messenger/main/server/docker-compose.yml
wget https://raw.githubusercontent.com/SharpRoma/Simple-Messenger/main/server/.env.example -O .env
```
Откройте файл `.env` любым текстовым редактором (например, `nano .env`) и **обязательно** замените стандартные пароли и секретные ключи на свои!

### 3. Запуск
Выполните команду для скачивания образов и запуска инфраструктуры в фоновом режиме:
```bash
docker compose up -d
```
Сервер автоматически поднимет PostgreSQL, создаст таблицы, зарегистрирует администратора и начнет слушать порт `8989`.

*(Все медиафайлы будут сохраняться в папку `./uploads` рядом с `docker-compose.yml` на вашем сервере).*

## Локальная разработка

Если вы хотите вносить изменения в код сервера и тестировать их на своем компьютере:

1. **Клонируйте репозиторий и создайте виртуальное окружение:**
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # Для Windows: .venv\Scripts\activate
   pip install -r requirements.txt
   ```

2. **Настройте переменные окружения:**
   Скопируйте `.env.example` в `.env`. Для локальной разработки установите:
   ```env
   DB_HOST=localhost
   ```

3. **Запустите локальную базу данных:**
   Вы можете поднять только PostgreSQL через Docker:
   ```bash
   docker compose up -d db
   ```

4. **Запустите FastAPI-сервер:**
   ```bash
   python run.py
   ```
   Сервер запустится с параметром `reload=True`. Любые изменения в коде мгновенно перезагрузят сервер.

## Документация API

FastAPI автоматически генерирует подробную документацию для всех REST-ручек.
После запуска сервера откройте в браузере:
**http://127.0.0.1:8989/docs**

Вы можете взаимодействовать с API прямо из браузера:
1. Раскройте секцию `POST /api/auth/register` и создайте пользователя.
2. Нажмите зеленую кнопку **Authorize** в правом верхнем углу и введите логин/пароль. Swagger автоматически добавит JWT-токен ко всем вашим следующим запросам.
3. Тестируйте загрузку файлов, создание чатов и удаление сообщений.

## Протокол WebSockets

Для мгновенной доставки сообщений клиент должен установить WebSocket соединение по адресу:
`ws://<IP_СЕРВЕРА>:8989/ws?token=<ВАШ_JWT_ТОКЕН>`

### Формат общения (JSON)
* **Отправка сообщения (Клиент -> Сервер):**
  ```json
  {
    "action": "send_msg",
    "chat_id": 1,
    "text": "Привет, мир!"
  }
  ```
* **Получение сообщения (Сервер -> Клиент):**
  ```json
  {
    "action": "new_msg",
    "chat_id": 1,
    "message": {
      "id": 42,
      "sender": "ivan",
      "text": "Привет, мир!",
      "file_name": null,
      "timestamp": 1715423891
    }
  }
  ```
