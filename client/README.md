# Simple Messenger — Клиентская часть

![Python](https://img.shields.io/badge/Python-3.12+-blue.svg)
![Flet](https://img.shields.io/badge/Flet-0.85+-ff69b4.svg)
![Platform](https://img.shields.io/badge/Platform-Windows%20%7C%20macOS-lightgrey.svg)
![Security](https://img.shields.io/badge/Security-Keyring-success.svg)

Кроссплатформенное графическое приложение, написанное на Python с использованием фреймворка [Flet](https://flet.dev/). 

Клиент взаимодействует с сервером по гибридной архитектуре: **REST API** (HTTP/HTTPS) используется для безопасной авторизации и потоковой передачи тяжелых файлов, а **WebSockets** (WS/WSS) обеспечивают мгновенную реалтайм-доставку сообщений.

---

## Архитектура (Модульная структура)

Клиент построен по принципу разделения ответственности и использует паттерны контроллеров (MVC). Интерфейс полностью отделен от сетевой логики, базы данных и системных вызовов ОС:

```text
client/
├── database.py            # Слой локальной БД SQLite с использованием SQLAlchemy ORM
├── system/                # Слой интеграции с ОС и вспомогательных модулей (Паттерн "Адаптер")
│   ├── base.py            # Базовый интерфейс адаптера
│   ├── macos.py           # Управление Dock-панелью и безопасные AppleScript уведомления
│   ├── windows.py         # Управление системным треем (pystray) и всплывающими окнами
│   ├── logger.py          # Настройка логирования с ротацией файлов
│   ├── factory.py         # Фабрика для инстанцирования ОС-специфичного адаптера
│   ├── crypto.py          # Модуль шифрования (RSA + AES-GCM) и безопасного хранения приватного ключа
│   ├── event_bus.py       # Асинхронная шина событий (EventBus) для связи сетевого слоя и UI
│   └── utils.py           # Общие кроссплатформенные системные утилиты ОС
├── ui/                    # Слой представления (UI Компоненты)
│   ├── main_window.py     # Тонкий фасад главного окна приложения (инициализирует AppController)
│   ├── controllers/       # Контроллеры бизнес-логики (AppController, AuthController, ChatController)
│   ├── screens/           # Изолированные экраны (Экран авторизации, Экран чата)
│   └── dialogs/           # Модальные окна (Настройки, Регистрация, Создание диалога)
├── assets/                # Графические ресурсы (иконки генерируются автоматически)
├── config.py              # Слой конфигурации: пути и настройки
├── network.py             # Сетевой слой: httpx (REST), websockets (Реалтайм) и авто-реконнект
└── main.py                # Точка входа: инициализирует логгер, адаптеры и графический движок
```

## Безопасность и Оптимизация

* **Защита паролей и приватных ключей:** Пароли и ключи не сохраняются на диске в открытом виде. Приложение интегрируется с системными хранилищами ключей (Связка ключей macOS / Диспетчер учетных данных Windows) через библиотеку `keyring`. Приватный RSA-ключ пользователя шифруется на диске с помощью AES-256-GCM по алгоритму `BestAvailableEncryption`, а сгенерированный 32-байтный пароль шифрования безопасно сохраняется в системном брелоке ОС.
* **Сквозное шифрование (E2EE) Секретных чатов:** Поддержка секретных чатов с шифрованием AES-GCM-256 для сообщений. RSA-2048 используется для безопасного обмена симметричными ключами. Сервер выступает в качестве zero-knowledge посредника, хранящего только зашифрованные ключи и ciphertext. Приватные ключи генерируются и хранятся исключительно локально на устройстве.
* **Локальное кэширование (SQLite на SQLAlchemy ORM):** Локальная БД SQLite переведена на SQLAlchemy ORM для лучшей типобезопасности и легкой поддержки схем. Она хранит зашифрованные метаданные и историю сообщений на доверенном клиентском устройстве (Telegram-style). При запуске интерфейс рендерится мгновенно из локальной копии, а новые сообщения догружаются в фоне с сервера. При этом поддерживается автоматическое добавление новых колонок при первом старте.
* **Локальный поиск:** Поиск по сообщениям и именам файлов в секретных чатах выполняется полностью локально на устройстве, сохраняя конфиденциальность переписки.
* **Отправка и скачивание файлов:** Файлы читаются напрямую с жесткого диска (Streaming / Multipart form-data), что позволяет отправлять большие файлы, не расходуя оперативную память компьютера.
* **Auto-Reconnect:** Встроенный отказоустойчивый механизм. При обрыве интернет-соединения клиент тихо уходит в фон, пытаясь восстановить подключение каждые 3 секунды, и автоматически подгружает пропущенные сообщения.
* **Шина событий (Event Bus):** Внедрение асинхронной шины событий (`EventBus`) позволило полностью избавить сетевой слой (`network.py`) от жесткой зависимости от компонентов графического интерфейса и UI-презентеров. Сетевые события транслируются в шину и потребляются соответствующими контроллерами.

## Нативные функции операционных систем

Приложение автоматически маскируется под нативную программу той ОС, в которой запущено:

| Функция | Windows | macOS |
| :--- | :--- | :--- |
| **Уведомления** | Нативные (интеграция через иконку в трее) | Нативные push-уведомления (AppleScript) |
| **Сворачивание** | Прячется в Системный трей | Элегантно сворачивается в Dock-панель |
| **Индикация** | Увеличенный красный бейдж на иконке в трее | Красный кружок в меню-баре / Dock |
| **Автозапуск** | Запись в Реестр `HKEY_CURRENT_USER\...\Run` | Создание LaunchAgent `.plist` |

## Запуск из исходного кода

1. Откройте терминал в корне проекта (на уровень выше папки `client`).
2. Создайте и активируйте виртуальное окружение:
   ```bash
   python -m venv .venv
   # Windows: .venv\Scripts\activate
   # macOS/Linux: source .venv/bin/activate
   ```
3. Установите зависимости:
   ```bash
   pip install -r client/requirements.txt
   ```
4. Запустите клиент:
   ```bash
   python client/main.py
   ```

5. Запустите юнит-тесты:
   ```bash
   # Выполните команду в корневой папке проекта
   PYTHONPATH=client .venv/bin/pytest client/tests
   ```

## CI/CD: Нативная сборка

Проект оснащен умными скриптами сборки, которые автоматически генерируют нативные установщики для конечных пользователей. 
**Единым источником правды** для версии приложения служит переменная `APP_VERSION` в файле `client/config.py`. Измените её, и скрипты сами обновят версию везде, включая защиту от даунгрейда.

### Для macOS (`scripts/build_mac.sh`)
Скрипт использует встроенный в Flet нативный macOS-компилятор (требует установленный Xcode).
* Автоматически собирает монолитное `.app` приложение (решает баг двух иконок в Dock).
* Прописывает версию в `Info.plist`.
* Упаковывает приложение в красивый установочный образ **`.dmg`** с помощью системной утилиты `hdiutil`.

### Для Windows (`scripts/build_win.bat`)
Скрипт автоматически компилирует `.exe` файл и интегрируется с **Inno Setup 6**.
* Собирает классический установщик `Setup.exe` со сжатием LZMA2.
* Создает ярлыки в меню Пуск и на Рабочем столе.
* Имеет встроенную защиту (Pascal Script), запрещающую установку старой версии поверх новой.

*Результаты сборки всегда помещаются в полностью очищенную папку `dist/` в корне проекта.*

## Логирование

В клиенте настроена автоматическая запись логов во время работы приложения. Логи пишутся как в стандартный поток вывода (консоль), так и в файл с ротацией.

* **Конфигурация:** [system/logger.py](file:///Users/sharproma/PycharmProjects/Simple-Messenger/client/system/logger.py).
* **Путь к файлу логов:** Файл логов `client.log` сохраняется в системной директории данных приложения:
  * **macOS:** `~/Library/Application Support/SimpleMessenger/logs/client.log`
  * **Windows:** `%APPDATA%\SimpleMessenger\logs\client.log` (обычно `C:\Users\<Имя_пользователя>\AppData\Roaming\SimpleMessenger\logs\client.log`)
  * **Linux:** `~/.config/SimpleMessenger/logs/client.log`
* **Параметры ротации:** При достижении размера в **5 МБ** лог автоматически архивируется. Клиент сохраняет до **3** архивных файлов (например, `client.log.1`, `client.log.2`).
* **Содержимое логов:** В лог записываются попытки подключения к серверу, сетевые ошибки, предупреждения от движка Flet и системные события адаптеров ОС.
