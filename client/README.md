# Simple Messenger — Клиентская часть

Этот модуль представляет собой кроссплатформенное графическое приложение (Desktop GUI), написанное на Python с использованием фреймворка [Flet](https://flet.dev/). Приложение взаимодействует с сервером по TCP-протоколу, используя асинхронные сокеты и SSL/TLS-шифрование.

---

## Архитектура (Модульная структура)

Клиент построен с использованием строгих паттернов проектирования, что позволяет легко менять дизайн, не затрагивая сетевую логику или системные вызовы:

* **`main.py`** — Точка входа. Инициализирует системные адаптеры и запускает UI.
* **`config.py`** — Слой данных. Управляет генерацией иконок и сохранением локальных настроек. Для максимальной безопасности пароли сохраняются не в JSON, а в системное хранилище (Связка ключей macOS / Диспетчер учетных данных Windows) через библиотеку `keyring`.
* **`network.py`** — Модель сети. Изолированный слой асинхронных сокетов (`asyncio`). Настраивает SSL-контекст, обеспечивает фоновый авто-реконнект (Auto-reconnect) при обрывах связи.
* **`system/` (Паттерн "Адаптер")** — Слой интеграции с ОС. Позволяет графическому интерфейсу вызывать системные функции (уведомления, трей, автозапуск), не задумываясь о том, на какой ОС он запущен:
  * `macos.py` — Управляет Dock-панелью и безопасными AppleScript уведомлениями (argv).
  * `windows.py` — Управляет системным треем (`pystray`).
* **`ui/` (Слой представления)** — Компоненты интерфейса:
  * `main_window.py` — Главный контроллер. Управляет состоянием приложения и переключением экранов.
  * `screens/` — Изолированные экраны (Авторизация, Чат).
  * `dialogs/` — Модальные окна (Настройки, Регистрация, Создание диалога).

## Нативные функции операционных систем

Приложение автоматически адаптируется под операционную систему:

| Функция | Windows | macOS |
| :--- | :--- | :--- |
| **Уведомления** | Нативные (через иконку в системном трее) | Нативные push-уведомления (AppleScript) |
| **Сворачивание** | Прячется в Системный трей | Сворачивается в Dock-панель |
| **Индикация** | Увеличенный красный бейдж на иконке в трее | Красный кружок в трее (меню-баре) |
| **Автозапуск** | Запись в Реестр `HKEY_CURRENT_USER\...\Run` | Создание LaunchAgent `.plist` |

## Запуск из исходного кода

Для запуска клиента в режиме разработчика:

1. Откройте терминал в корне проекта (на уровень выше папки `client`).
2. Создайте и активируйте виртуальное окружение:
    ```Bash
    python -m venv .venv
    # Windows: .venv\Scripts\activate
    # macOS/Linux: source .venv/bin/activate
    ```
3. Установите зависимости:
    ```Bash
    pip install -r client/requirements.txt
    ```
4. Запустите клиент:
    ```Bash
    python client/main.py
    ```
