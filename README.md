**Simple Messenger** — это современный, высоконагруженный и защищенный кроссплатформенный мессенджер с клиент-серверной архитектурой. Создан на Python с использованием следующих технологий: **FastAPI + PostgreSQL** для бэкенда и **Flet** для графического интерфейса.

Проект подходит для развертывания в корпоративной локальной сети (LAN) или на личном VPS (WAN) без зависимости от сторонних облачных сервисов.

## Ключевые возможности

* **Гибридная архитектура сети:** Совмещение REST API (авторизация, история, передача файлов) и мгновенных WebSockets (реалтайм-доставка сообщений и уведомлений).
* **Стриминг файлов:** Передача медиафайлов происходит чанками. Загрузка и скачивание файлов любого размера практически не расходует оперативную память сервера и клиента.
* **Безопасность и Шифрование:** Весь трафик между клиентом и сервером защищен SSL/TLS. Авторизация построена на JWT-токенах, пароли в БД хешируются (`bcrypt`), а пароли на клиентах надежно спрятаны в системные хранилища ОС (Keyring / Keychain).
* **Ролевая модель:** Встроенная система прав с автоматическим созданием суперпользователя (`admin`), который имеет доступ к удалению любых сообщений.
* **Нативная интеграция с ОС (Адаптеры):**
  * **macOS:** Нативное сворачивание в Dock, безопасные системные уведомления AppleScript без риска инъекций.
  * **Windows:** Сворачивание в системный трей (`pystray`) с поддержкой динамических бейджей непрочитанных сообщений.
* **CI/CD:** Автоматическая сборка готовых к использованию нативных установщиков (`.dmg` для macOS и `Setup.exe` через Inno Setup для Windows) с защитой от установки старых версий.

## Структура проекта

Проект разделен на две независимые модульные части:

* **[`/server`](./server/)** — Высокопроизводительный бэкенд (FastAPI + SQLAlchemy 2.0 + PostgreSQL). Автоматически генерирует интерактивную Swagger-документацию. Упакован в Docker и публикуется в GitHub Container Registry (GHCR) для развертывания в один клик. Подробная инструкция находится в [**Документации сервера**](./server/README.md).
* **[`/client`](./client/)** — Графическое Desktop-приложение. Построено по строгой компонентной архитектуре (MVC/MVVM). Подробное описание устройства клиента и сборки инсталляторов находится в [**Документации клиента**](./client/README.md).

## Быстрый запуск сервера

Благодаря использованию GitHub Container Registry, вам не нужно копировать исходный код на ваш VDS. Достаточно создать файл `.env` с вашими паролями и запустить `docker-compose.yml`:

```bash
docker compose up -d
```
Сервер автоматически скачает готовый образ бэкенда, поднимет СУБД PostgreSQL, сгенерирует SSL-сертификаты и начнет работу. Подробнее читайте в [документации сервера](./server/README.md).
